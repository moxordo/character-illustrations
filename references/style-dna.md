# Style DNA (shared across all characters)

## One sentence

Pure white, minimal, hand-drawn, lots of whitespace, restrained, deadpan, slightly absurd, product-sketch feeling — structure clear but never an instruction manual. Like someone who has spent years on AI, product, and dev tools quickly sketching one explanatory drawing on a blank sheet of paper.

## Must have

- 16:9 horizontal concept illustration.
- Pure white background: no beige, no warm gray, no paper texture, no gradient, no shadow, no noise, no vintage feel.
- Mostly black hand-drawn line art: thin lines, slightly wobbly, not mechanical, not vector-perfect, not heavy outlines.
- Lots of whitespace: the main subject fills about 40%–60% of the canvas, at least 35% stays empty, ideally one whole quiet zone.
- One image expresses only one core action, structure, state, or metaphor.
- Let structure read naturally — never write the structure type name on the image, and never put a type-title in the top-left corner.

## Palette modes

The active character profile's frontmatter picks the mode and the exact hexes.

**single-accent** — black plus exactly one accent color. The accent does one job per image: the main flow, the key arrow, the critical word, the result, or the moment of emphasis. Never introduce a second color; if tempted, make it the accent or make it black.

**semantic-triad** — black plus three accents with fixed meanings:

- **orange** — main flow, paths, arrows, automation direction, A→B movement.
- **red** — key callouts, problems, emotional beats, warnings, results.
- **blue** — secondary notes, brain/system state, second-layer explanation, AI/assistant/meta hints. Blue is optional per image.

In both modes: color is restrained. Fewer is better. Black carries structure, character, frames, and main text.

## Labels

- Handwritten feel, short: 1–3 words (or 2–8 characters in CJK) each.
- Baseline budget 5–8 labels per image. **Exception:** actor labels (see composition-patterns.md) — every actor gets named even if that exceeds the budget.
- Language: match the input content's language, unless the character profile pins one (`label_language`).
- Never let text become paragraph-length explanation.

## Never

- No commercial illustration, no polished flat illustration.
- No PPT infographic, no course slide, no formal flowchart.
- No cute cartoon poster, no children's illustration.
- No complex architecture diagram; do not explain every single node.
- No techy UI, no realistic app screenshot.
- No complex background, gradient, shadow, or texture.
- No off-palette colors.
- No type-title in the top-left corner ("Workflow / System Diagram / Common Pitfalls / Roadmap", etc.).

## Aesthetic direction

Aim for deadpan, inventive, interesting, clean, and a little absurd.
Avoid cute, childish, over-complicated, or stiff.

A high-quality image makes the reader think "that's a bit odd" first, then read the structure within one second. If the first glance says "tutorial page" instead of "strange product sketch on blank paper", it fails.
